# PROJECT_RULES.md

# Logic Coach V1

## Objectif

Développer une application Symfony 7.4 LTS simple, robuste et professionnelle permettant à un utilisateur de suivre quotidiennement son bien-être grâce à un questionnaire et des conseils adaptés.

---

# Technologies

- Symfony 7.4 LTS
- PHP 8.3
- Doctrine ORM
- Twig
- MySQL
- PHPUnit
- Git
- GitHub

---

# Règles de développement

- Respecter les bonnes pratiques Symfony.
- Respecter l'architecture MVC.
- Utiliser Doctrine ORM pour tous les accès à la base de données.
- Écrire un code simple, lisible et maintenable.
- Ne jamais dupliquer le code.
- Commenter uniquement lorsque cela apporte une réelle valeur.
- Tester chaque fonctionnalité avant le commit.

---

# Git

Chaque fonctionnalité terminée doit respecter l'ordre suivant :

1. Vérification
2. Tests
3. Commit
4. Push GitHub

---

# Documentation

Chaque séance produit :

- une évolution du projet ;
- une documentation technique dans le dossier `docs`.

---

# Objectif final

Obtenir un projet entièrement fonctionnel, documenté et prêt à être présenté devant le jury.
